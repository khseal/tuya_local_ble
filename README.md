# Raykube A1 Ultra (hc7n0urm) Local Control for Home Assistant

⚠️ **Это решение предназначено для Raykube A1 Ultra (`hc7n0urm`).\
Это НЕ A1 PRO.**

------------------------------------------------------------------------

## О проекте

За основу был взят репозиторий:
https://github.com/khseal/tuya_local_ble/

С большой помощью GPT удалось отреверсить протокол и запустить
**локальное управление замком через BLE без облака**.

Какие ошибки ещё могут быть в коде --- пока неизвестно, но на данный
момент у меня всё работает.

Файл конфигурации:
    /config/tuya_local_ble/devices.json

Пример содержимого:
``` json
{
  "DC:23:77:B2:99:99": {
    "address": "DC:23:77:B2:99:99",
    "uuid": "8c9715f859c5ab27",
    "local_key": "YTKMRPY+UhexW7Vw",
    "device_id": "bf85b9ar78U5lK87",
    "category": "jtmspro",
    "product_id": "hc7n0urm",
    "device_name": "A1 Ultra",
    "product_model": "",
```

# Где взять параметры устройства

Часть параметров можно получить через **Tuya Developer Portal**.
Видео с примером получения данных:
https://www.youtube.com/watch?v=2bL7OGSVzLw

Оттуда берутся поля:
    uuid
    local_key
    device_id
    category
    product_id
    device_name
    product_model
    product_name

# Unlock Token

Для открытия замка требуется дополнительный параметр:
    unlock_token

Это **некий внутренний код**, который используется приложением при
BLE‑разблокировке.

Где его взять и является ли он постоянным --- точно неизвестно.
Для моего замка он был извлечён из **Bluetooth HCI snoop log** телефона.
Полезная информация о том, как получить HCI лог:
https://stackoverflow.com/questions/28445552/bluetooth-hci-snoop-log-not-generated

Пример:
    "unlock_token": "12742760"

# Установка

## Option 2: Manual
Поместите папку интеграции в директорию `custom_components`:

    config/
    └── custom_components/
        └── tuya_ble_local/
        
А файл конфигурации:

    config/
    └── tuya_local_ble/
        └── devices.json

После этого перезапустите **Home Assistant**.
В Меню Настройки --> Устройства и Службы --> Интеграции должно появиться предложение, 
добавить устройство через интеграцию Tuya Local BLE. 
Ручного добавления через меню не предусмотрено.
